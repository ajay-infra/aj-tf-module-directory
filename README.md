# aj-tf-module-directory

Terraform module for AWS Managed Microsoft AD in the management VPC. Part of L2 in the platform stack ŌĆö provisioned once per account, before the Client VPN endpoint.

---

## What this module does

Provisions **AWS Managed Microsoft AD** (`MicrosoftAD` type) in the management VPC and wires it up for use:

- **Directory** ŌĆö `Standard` or `Enterprise` edition, two AZs, admin password auto-generated
- **Admin credentials** ŌĆö stored in Secrets Manager immediately; never in tfvars or state in plaintext
- **Security group** ŌĆö all AD-required ports (DNS, Kerberos, LDAP, LDAPS, SMB, Global Catalog) open to `allowed_cidrs`
- **DHCP options** ŌĆö points management VPC DNS to AD servers so VPN-connected engineers resolve `corp.*` names

The directory ID and DNS IPs flow directly into `aj-tf-module-vpn` as variables.

---

## Why a management VPC

AD sits in its own management VPC (`10.200.0.0/16` or `10.201.0.0/16`) ŌĆö completely separate from dev/staging/prod workload VPCs. Engineers reach it through Client VPN, not by direct network access from clusters.

```
Engineer laptop
  ŌåÆ Client VPN endpoint (aj-tf-module-vpn)
    ŌåÆ management VPC
      ŌåÆ AWS Managed AD (this module)
        ŌåÆ authenticates VPN session
        ŌåÆ resolves corp.platform.internal DNS
```

---

## Apply order

Directory is **Stage N1** ŌĆö after the management VPC, before VPN:

```
Stage C1: aj-tf-module-vpc   ŌåÆ management VPC (10.200.0.0/16)
Stage N1: aj-tf-module-directory  ŌåÉ this module
            ŌåÆ directory_id ŌåÆ passed to aj-tf-module-vpn
            ŌåÆ dns_ip_addresses ŌåÆ passed to aj-tf-module-vpn
Stage N2: aj-tf-module-vpn   ŌåÆ Client VPN endpoint
```

Applied once per account tier (central-nonprod manages dev+staging; central-prod manages prod).

## Usage

```hcl
module "directory" {
  source = "github.com/ajay-infra/aj-tf-module-directory?ref=v0.1.0"

  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_subnet_ids

  domain_name       = "corp.platform.internal"
  domain_short_name = "CORP"
  allowed_cidrs     = ["172.16.0.0/22", "10.200.0.0/16"]

  edition     = "Standard"
  environment = "dev"
}

# Pass outputs to the VPN module
module "vpn" {
  source = "github.com/ajay-infra/aj-tf-module-vpn?ref=v0.1.0"

  directory_id  = module.directory.directory_id
  dns_ip_addrs  = module.directory.dns_ip_addresses
  ...
}
```

## Admin password

The admin password is generated by Terraform and stored in Secrets Manager at `<environment>/directory/admin-password`. It is never written to tfvars, never in version control, and marked `sensitive` so it does not appear in plan output.

```json
{
  "username":     "Admin",
  "password":     "<generated-32-char>",
  "domain":       "corp.platform.internal",
  "domain_short": "CORP"
}
```

Use this for **break-glass access only** ŌĆö day-to-day operations use IAM Identity Center (SSO) or role-based AD accounts.

## Standard vs Enterprise edition

| | Standard | Enterprise |
|---|---|---|
| Max objects | 30,000 | 500,000 |
| AD trusts | No | Yes |
| Schema extensions | No | Yes |
| Approx. monthly cost | ~$146 | ~$288 |
| Use for | Client VPN auth, small org | Large org, cross-domain trusts |

Use `Standard` unless you need trusts or very large object counts.

## AD ports opened in the security group

| Port | Protocol | Service |
|---|---|---|
| 53 | TCP + UDP | DNS |
| 88 | TCP + UDP | Kerberos authentication |
| 389 | TCP + UDP | LDAP |
| 445 | TCP | SMB / AD replication |
| 464 | TCP + UDP | Kerberos password change |
| 636 | TCP | LDAPS (LDAP over SSL) |
| 3268 | TCP | Global Catalog |
| 3269 | TCP | Global Catalog SSL |

Source is `allowed_cidrs` ŌĆö set to your Client VPN client CIDR + management VPC CIDR. Do not open to `0.0.0.0/0`.
